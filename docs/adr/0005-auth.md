# ADR 0005 — Autentizace (M1)

**Status:** Accepted · **Datum:** 2026-06-14

## Kontext

M1 potřebuje účty a ochranu herních endpointů. Singleplayer-first, ale identita je nutná i pro pozdější MP.

## Rozhodnutí

- **Účet:** username + heslo (email volitelný). Hesla hashovaná **bcryptjs** (čisté JS → přenositelné na alpine/CI bez nativního buildu; argon2 zavrženo kvůli nativní kompilaci).
- **Tokeny:** JWT **access** (krátký, default 15m) + **refresh** (default 30d), oba podepsané `JWT_SECRET`. Stateless (žádná server-side session) → škálovatelnost (ADR 0003).
- **Guard:** vlastní `JwtAuthGuard` (bez passportu) ověří `Authorization: Bearer` a naplní `req.user`.
- **Token na webu:** zatím `localStorage` + Authorization header; refresh při 401 jednorázově.

## Follow-up (vědomý dluh)

- **httpOnly cookie** místo localStorage (ochrana proti XSS) — přesunout, až bude SSR auth flow.
- **Refresh rotace + revokace** (uložení jti do Redisu, blacklist na logout).
- `JWT_SECRET` v produkci MUSÍ přijít z env (default je jen pro lokální dev).

## Důsledky

- Endpointy se chrání `@UseGuards(JwtAuthGuard)`; `@CurrentUser()` dává `{ accountId, username }`.
- Nové herní moduly nemají řešit auth — jen použít guard + dekorátor.
